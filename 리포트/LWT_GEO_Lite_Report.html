FINDING 01 SPA 콘텐츠 미노출 긴급

AI 크롤러가 우리 사이트에서 한 글자도 읽지 못한다.

정적 HTML 3,058 bytes · H1 태그 0/20 페이지 · 본문 텍스트 0%

우리 사이트는 JavaScript로 화면을 그리는 SPA(Single Page Application) 구조입니다. 사용자가 브라우저로 접속하면 멋진 화면이 보이지만, AI 크롤러가 처음 페이지를 받아왔을 때 그 안에는 단 3,058 bytes(A4 한 페이지의 1/10 분량)의 코드만 들어 있고, 정작 "이 페이지가 뭐에 관한 페이지인지" 알려주는 본문 텍스트는 없습니다. AI 입장에서는 "주소만 있고 본문이 비어있는 편지"를 받는 셈입니다.
현황 · AS-IS
  • 정적 HTML 크기 3,058 bytes (메타 데이터 + 스크립트 로더만 포함)
  • 본문 콘텐츠는 /assets/client.js + /assets/main.js 로드 후 클라이언트에서 동적 생성
  • H1 태그 0/20 페이지, 이미지 alt 텍스트 0개 (정적 HTML 기준)
  • AI 크롤러가 페이지 주제를 메타 디스크립션 1~2줄로만 추측해야 하는 상태
  • "이 페이지가 명품 쇼핑 가이드인지, 아쿠아리움 안내인지"를 AI가 텍스트로 확인 불가
처방 · TO-BE
  • SPA 구조 변경 없이 <body> 최상단에 정적 콘텐츠 블록 인라인 삽입 (display:none + aria-hidden 처리하여 화면 영향 없음)
  • 페이지당 H1 1개 + 핵심 텍스트 300자 이상 + 주요 시설·브랜드 명칭 노출
  • 중장기로 SSR(Server-Side Rendering) 또는 프리렌더링 도입 검토
  • 예상 효과: AI 크롤러 페이지 주제 인식률 0% → 100%
  • ※ 즉시 적용 가능한 수정 코드는 STANDARD에서 제공됩니다.
FINDING 02 Schema.org 0% 긴급

"우리가 어떤 회사인지" 알려주는 구조화 데이터가 0%다.

Schema.org JSON-LD 0/20 페이지 · Organization · LocalBusiness · Restaurant · Product 전부 미적용

Schema.org는 사이트의 정보를 AI와 검색엔진이 이해하기 쉬운 "표준 양식"으로 정리해주는 코드입니다. "이 페이지는 음식점입니다 / 가게 이름은 X / 영업시간은 Y / 가격대는 Z" 같은 정보를 명함처럼 정리해서 건네는 셈입니다. 우리 사이트에는 이 명함이 한 장도 없습니다. AI는 우리가 쇼핑몰인지 호텔인지 박물관인지조차 코드 레벨에서는 알 수 없는 상태입니다.
현황 · AS-IS
  • 크롤링한 20개 페이지 전체에서 JSON-LD 구조화 데이터 0개 발견
  • GEO 필수 @type 모두 미적용: Organization, LocalBusiness, ShoppingCenter, Place, Restaurant, Product, Event, FAQPage, BreadcrumbList
  • 업종 평균 적용률: 약 35~60% (동종 복합쇼핑몰 기준)
  • AI 엔진이 주소·영업시간·시설·브랜드 정보를 정형화된 형태로 인식 불가
처방 · TO-BE
  • 1순위 (Day 1): Organization + LocalBusiness/ShoppingCenter + FAQPage 메인 페이지 적용
  • 2순위 (Week 1): 브랜드 목록 페이지에 Product, 식당 가이드에 Restaurant, 이벤트 페이지에 Event 스키마 적용
  • 3순위 (Week 2): 시설별 페이지에 Place + BreadcrumbList로 사이트 계층 구조 전달
  • 예상 효과: GEO 점수 A 카테고리 0/20 → 15/20 회복
  • ※ 즉시 적용 가능한 수정 코드는 STANDARD에서 제공됩니다.
FINDING 03 콘텐츠 의도 갭 높음

FAQ 페이지 부재로 "문제해결형 질문"에서 우리가 답이 될 수 없다.

FAQ 페이지 0개 · HowTo 콘텐츠 0개 · 자주 묻는 질문 공식 답변 부재

"[자사 권역]에서 어린이랑 시간 보낼 만한 곳", "외국인 택스리펀 어떻게 받나요" 같은 문제해결형 질문은 AI가 가장 자주 답하는 유형입니다. 그런데 우리 사이트에는 이런 질문에 직접 답해주는 FAQ 페이지가 없습니다. 그래서 AI는 어쩔 수 없이 외부 블로그·여행 후기에서 답변을 가져옵니다. 우리는 시설과 서비스를 보유하고 있는데, AI는 그 사실을 모른 채 답하는 답답한 상황입니다.
현황 · AS-IS
  • 사이트 내 FAQ 페이지 자체가 존재하지 않음 — 자주 묻는 질문에 대한 공식 답변 0개
  • HowTo·가이드 콘텐츠 부재 — 어린이 방문·외국인 가이드·교통 안내 등 페르소나별 콘텐츠 없음
  • 영업시간·주차·교통·시설 안내는 메타 디스크립션 1~2줄로만 존재
  • 비교 분석 콘텐츠 부재 — "○○○쇼핑몰 vs [경쟁몰 가]" 같은 비교형 질문에서 자사 인용 불가
처방 · TO-BE
  • FAQ 페이지 신설 + FAQPage 스키마 적용 — 영업시간·주차·교통·키즈 시설·외국인 서비스·예약 등 Q&A 20개 이상
  • "어린이와 함께" "외국인 가이드" 등 페르소나별 정적 콘텐츠 페이지 신설
  • 비교 콘텐츠 정기 발행 — 자사가 비교형 질문의 1차 출처가 되도록
  • 예상 효과: 문제해결형 매칭률 +30%p, 자사 인용률 +10%p
  • ※ AI 엔진 실측 결과와 카테고리별 매칭 분석은 STANDARD에서 제공됩니다.
